hexo 指令整理

常用的指令→ 寫完文章，hexo clean, hexo g , hexo d

指令 說明
hexo init [folder] 建立一個新的網站
hexo new [name] [title] 創建新文章
hexo generate 產生靜態檔案, hexo g 也可
hexo deploy 部署網站，就是生成網站了
hexo deploy -g 部署前先生成靜態擋案
hexo clean 清除快取檔案 (db.json) 和已產生的靜態檔案 (public)。
hexo –debug 在終端機中顯示除錯訊息並儲存記錄檔到 debug.log。當您碰到問題時，試著以除錯模式重新執行一次，並 把除錯訊息貼到 GitHub。
hexo –draft 顯示草稿

hexo 文章的asset資料夾

  1. 首先必須先在 _config.yml 中設定「post_asset_folder」為true

  1. 以上設定完之後，建立新文章會自動生出與新文章相同名稱之資料夾
    ※ 為了避免錯誤，建立的文章會以英文為主。

  2. 引用圖片(assets)的方式：
